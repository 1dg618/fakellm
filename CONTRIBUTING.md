# Contributing to FakeLLM

Thanks for your interest in contributing! Contributions of all kinds are welcome — bug reports, fixes, features, docs, and ideas.

## Quick Start

1. Fork the repo and clone your fork
2. Create a branch: `git checkout -b my-change`
3. Make your changes and test them
4. Push to your fork and open a pull request against `main`

## Reporting Bugs

Open an issue with:
- What happened vs. what you expected
- Steps to reproduce
- Your Python version and OS

## Suggesting Features

Open an issue describing the feature and the use case. Happy to discuss before you write any code.

## Pull Requests

- Keep changes focused — one PR per logical change
- Add tests if you're adding or fixing behavior
- Update the README if you change how something works
- Be patient — I'll review as soon as I can

## Code of Conduct

By participating, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).

---

Not sure where to start? Check the [issues](https://github.com/1dg618/fakellm/issues) for anything labeled `good first issue`, or just open an issue to introduce yourself.
